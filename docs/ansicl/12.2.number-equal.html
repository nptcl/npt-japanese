Function =, /=, <, >, <=, >=

構文

= &rest number+ => generalized-boolean
/= &rest number+ => generalized-boolean
< &rest number+ => generalized-boolean
> &rest number+ => generalized-boolean
<= &rest number+ => generalized-boolean
>= &rest number+ => generalized-boolean

引数と戻り値

number - <, >, <=, >=は実数。 =, /=は数。
generalized-boolean - generalized-boolean

定義

=, /=, <, >, <=, >=は、 次のようにそれぞれの引数に対して数学的な比較を行います。

  • =
    • もし全てのnumberが同じ値のとき、=の値はtrueであり、 それ以外のときはfalseです。 複素数のときは、実部と虚部が=で等しいときに その2つの複素数が=で等しいと考えます。
  • /=
    • もしどれも2つのnumberが等くないとき、/=の値はtrueであり、 それ以外のときはfalseです。
  • <
    • もし全てのnumberが単調増加の順番のとき、<の値はtrueであり、 それ以外のときはfalseです。
  • >
    • もし全てのnumberが単調減少の順番のとき、>の値はtrueであり、 それ以外のときはfalseです。
  • <=
    • もし全てのnumberが単調減少ではない順番のとき、<=の値はtrueであり、 それ以外のときはfalseです。
  • >=
    • もし全てのnumberが単調増加ではない順番のとき、>=の値はtrueであり、 それ以外のときはfalseです。

=, /=, <, >, <=, >=は、 必要に応じて型の変換を行います。

(< 3 -5) は false.            (<= 3 -5) は false.
(< 3 3) は false.             (<= 3 3) は true.
(< 0 3 4 6 7) は true.        (<= 0 3 4 6 7) は true.
(< 0 3 4 4 6) は false.       (<= 0 3 4 4 6) は true.
(> 4 3) は true.              (>= 4 3) は true.
(> 4 3 2 1 0) は true.        (>= 4 3 2 1 0) は true.

例外

引数のどれかが実数ではないとき、 型type-errorを通知するかもしれません。 その他に比較を実施することができないときは 型arithmetic-errorが通知されるかもしれません

備考

=はeqlとはちがい、 (= 0.0 -0.0)はいつもtrueです。 なぜなら=はそれらの引数は 数学的な値として比較されるからであり、 一方eqlは 言うならば表現による値で比較します。
